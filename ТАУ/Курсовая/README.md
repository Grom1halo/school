# ЧАСТЬ 1 — ТЕОРИЯ (что нужно знать на защите)

## 1.1 Что за объект управления

Бесцентровое шлифование — это когда заготовку шлифуют без зажима в центрах. Деталь
подаётся между шлифовальным кругом (ШК) и ведущим кругом (ВК). Припуск `h` — толщина
снимаемого слоя металла.

**Задача системы:** поддерживать мощность резания `N` постоянной, несмотря на то что
припуск `h` меняется (разные заготовки, износ инструмента).

**Почему мощность нужно стабилизировать:**
- Слишком большая мощность → перегрев, прижоги, брак
- Слишком маленькая → производительность падает

---

## 1.2 Математическая модель объекта

Объект — это цепочка:

```
Задающее напряжение U(p)
        ↓
   Двигатель подачи
   W_дв(p) = 1/C1 / (T1·Tm·p² + Tm·p + 1)
        ↓
   Линейная скорость S = k1·ω
        ↓
   Мощность резания N1 = k2·S·h   ← здесь h — параметрическое возмущение!
        ↓
   Привод шлифовального круга
   W_шк(p) = 1/(T2·p + 1)
        ↓
   Выход N2(p)
```

**Итоговая передаточная функция объекта:**

```
              K(h)
W_obj(p) = ──────────────────────────────────
           (T1·Tm·p² + Tm·p + 1)·(T2·p + 1)
```

где `K(h) = k1·k2·h / C1` — статический коэффициент, зависит от припуска.

**Параметры (вариант 5):**

| Параметр | Значение | Физический смысл |
|----------|----------|-----------------|
| R1 = 50 Ом | сопротивление якорной цепи |
| T1 = 0.4 с | постоянная времени якорной цепи (T1 = L/R1) |
| C1 = C2 = 11 | ДЭДС и коэффициент момента двигателя |
| J = 0.06 кг·м² | момент инерции ротора |
| Tm = J·R1/(C1·C2) ≈ 0.02479 с | электромеханическая постоянная |
| T2 = 0.07 с | постоянная привода ШК |
| k1 = 5.3·10⁻³ м | коэффициент скорости подачи |
| k2 = 1.3·10⁸ Н/м | коэффициент силы резания |
| Ku = 13.64·10⁻³ | коэффициент датчика |
| K3 = 0.003 с | постоянная времени датчика |

**Порядок системы:** 3 (два полюса от двигателя + один от привода ШК).

## 1.3 Почему объект «сложный»

Знаменатель квадратного звена `T1·Tm·p² + Tm·p + 1`:

```python
# из sim.py:
Tm  = J * R1 / (C1 * C2)   # ≈ 0.02479 с

# Коэффициент демпфирования:
# T1·Tm·p² + Tm·p + 1 ≡ p²/ωn² + 2ξ/ωn·p + 1
# => ωn = 1/√(T1·Tm) = 1/√(0.4·0.02479) ≈ 10.04 рад/с
# => ξ = Tm/(2·√(T1·Tm)) = 1/(2·ωn·T1) ≈ 0.124
```

`ξ = 0.124 < 0.5` — **слабодемпфированное колебательное звено.**

Это значит: без хорошего регулятора система будет долго колебаться перед тем,
как успокоиться. На ЛАЧХ это виден как резкий горб около `ωn = 10 рад/с`.

## 1.4 Что такое ПИД-регулятор и зачем он здесь

```
W_рег(p) = Kp · (1 + 1/(Ti·p) + Td·p)
```

| Канал | Действие | Зачем нужен здесь |
|-------|----------|-------------------|
| P — пропорциональный | мгновенно реагирует на ошибку | масштабирование усиления |
| I — интегральный (1/Ti·p) | накапливает ошибку, убирает статическую | **нулевая статическая ошибка** δ = 0 |
| D — дифференциальный (Td·p) | реагирует на скорость изменения ошибки | **гашение колебаний** объекта (ξ = 0.124) |

## 1.5 Частотный метод синтеза (главное на защите!)

Это метод по Французовой. Суть в 4 шагах:

### Шаг 1 — Строим ЛАЧХ объекта

ЛАЧХ = логарифмическая амплитудно-частотная характеристика:
```
L(ω) = 20·lg|W(jω)|   [дБ]
```

На графике (`bode_plant.png`) видно три кривые для h_min, h_nom, h_max.
Они отличаются только по вертикали (K(h) разный), форма одинакова.

**Характерные частоты на ЛАЧХ объекта:**
- `1/T1 = 2.5 рад/с` — перегиб из-за апериодического звена якорной цепи
- `ωn = 10.04 рад/с` — резонанс колебательного звена (горб)
- `1/T2 = 14.3 рад/с` — перегиб из-за привода ШК
- `1/Tm = 40.3 рад/с` — ещё один перегиб (высокая частота)

### Шаг 2 — Формулируем требования

```
tп ≤ 0.6 с  →  нужна частота среза ωc ≈ 4/tп = 4/0.6 ≈ 6.67 рад/с
σ ≤ 0.6%    →  запас по фазе φ_зап ≥ 60° (монотонный процесс)
```

### Шаг 3 — Строим желаемую ЛАЧХ

Желаемая ЛАЧХ — это то, какой должна быть ЛАЧХ разомкнутой системы,
чтобы переходный процесс был таким, как нам нужно.

Правило: **наклон −20 дБ/дек в районе ωc** обеспечивает φ_зап ≥ 60°.

```python
# из lachx_synthesis.py:
K_open_desired = wc           # условие: L_ж(wc) = 0 дБ
L_desired = 20 * np.log10(K_open_desired / w)  # наклон -20 дБ/дек
```

### Шаг 4 — ЛАЧХ регулятора = разность

```
L_рег(ω) = L_ж(ω) − L_obj(ω)
```

Смотрим на форму `L_рег(ω)` и «узнаём» в ней ПИД:

```
Нуль L_рег на ω = 1/T1 = 2.5 рад/с   →  Td = T1 = 0.4 с
Нуль L_рег на ω = 1/Tm = 40.3 рад/с  →  Ti = Tm = 0.02479 с
Наклон +20 дБ/дек при низких ω       →  интегрирующее звено (I-канал)
```

**Это и есть главный вывод:** параметры Ti и Td найдены не перебором,
а из анализа ЛАЧХ. Регулятор «компенсирует» перегибы ЛАЧХ объекта.

### Шаг 5 — Выбор Kp

Kp подбирается из условия единичного пересечения ЛАЧХ разомкнутой системы
на частоте среза ωc = 4.5 рад/с:

```
|W_рег(jωc)·W_obj(jωc)·W_датч(jωc)| = 1
```

Отсюда `Kp = 0.7643`.

## 1.6 Статическая точность

Требование: `δ = 0.6%` (в условии задания).

**Фактически δ = 0** — потому что в регуляторе есть I-канал (интегратор).

Интегратор означает, что при постоянном задании разомкнутая система имеет
астатизм 1-го порядка → установившаяся ошибка = 0 автоматически.

*Требование δ ≤ 0.6% выполнено с запасом.*

---

## 1.7 Почему расчёт ведётся при h_max

Из методики Французовой (раздел 4.2): проектируем для **наихудшего случая**.

При `h_max = 180 мкм`: K(h) максимальный → система наиболее склонна к
нестабильности. Если регулятор обеспечивает σ ≤ 0.6% при h_max — при
меньших h он тем более справится (с нулевым σ, просто чуть медленнее).

| Припуск | tп | σ | Норма? |
|---------|----|---|--------|
| h_max = 180 мкм | 0.452 с | 0.559% | ✓ tп ≤ 0.6 с |
| h_nom = 125 мкм | 0.726 с | 0.000% | монотонный, брака нет |
| h_min = 70 мкм  | 1.458 с | 0.000% | медленный, но безопасный |

## 1.8 Запасы устойчивости

- **Запас по фазе** φ_зап = 71.7° ≥ 60° ✓
- **Запас по амплитуде** > 6 дБ ✓

## 2.2 Ключевые функции

### plant_tf(h) — передаточная функция объекта

```python
def plant_tf(h):
    K  = K_obj(h)                    # K(h) = k1*k2*h / C1
    num = [K]                        # числитель: просто K
    d1 = [T1 * Tm, Tm, 1.0]         # квадратный полином (колебательное звено)
    d2 = [T2, 1.0]                   # апериодическое звено (привод ШК)
    den = np.polymul(d1, d2)         # перемножаем: получаем полином 3-го порядка
    return signal.TransferFunction(num, den)
```

Результат: `W_obj(p) = K / [(T1·Tm·p² + Tm·p + 1)·(T2·p + 1)]`

### closed_loop_step() — замкнутая система

```python
# Разомкнутая: W_ol = W_pid * W_plant * W_sens
ol_num = Cnum * Gnum * Snum   # перемножаем числители
ol_den = Cden * Gden * Sden   # перемножаем знаменатели

# Замкнутая по теореме: W_cl = W_ol / (1 + W_ol)
#   числитель = C*G (без датчика в прямой цепи)
#   знаменатель = Cden*Gden*Sden + Cnum*Gnum*Snum
```

Это стандартная формула замкнутой системы с единичной ОС.

### quality_metrics() — показатели качества

```python
yn = y / y[-1]                      # нормируем переходную характеристику
sigma = (max(yn) - 1.0) * 100       # перерегулирование в %
# tп = момент последнего выхода за полосу ±5%
outside = where(|yn - 1| > 0.05)
tp = t[outside[-1]]
```

### cost_function() — что оптимизируется

```python
J = (штраф за tп > 0.6) * 200       # если tп превышает — большой штраф
  + (штраф за σ > 0.6%) * 8000      # за перерегулирование штраф ещё больше
  + tп * 8                           # стремимся к быстроте
```

Именно эта функция объясняет, почему алгоритм нашёл Ti ≈ Tm, Td ≈ T1:
они дают минимальное tп при σ → 0.

# ЧАСТЬ 3 — ВОПРОСЫ НА ЗАЩИТЕ И ОТВЕТЫ

**Q: Что такое Tm и откуда она берётся?**

A: Tm = J·R1/(C1·C2) — электромеханическая постоянная двигателя. Это время,
за которое двигатель разгоняется до номинальной скорости при номинальном
напряжении без нагрузки. Здесь Tm ≈ 0.0248 с — очень маленькая (быстрый двигатель).

---

**Q: Почему именно Ti = Tm и Td = T1?**

A: Из анализа ЛАЧХ регулятора (рис. 3.1). ЛАЧХ регулятора = желаемая − ЛАЧХ объекта.
На полученном графике видны нули на частотах 1/T1 = 2.5 рад/с и 1/Tm = 40.3 рад/с —
это и есть Td и Ti ПИД-регулятора. Физически: регулятор «компенсирует» перегибы
ЛАЧХ объекта на этих частотах.

---

**Q: Почему tп при h_nom = 0.726 с, хотя требование 0.6 с?**

A: По методике (Французова, раздел 4.2) синтез ведётся для наихудшего случая
h = h_max. При h_max достигнуто tп = 0.452 с ≤ 0.6 с — требование выполнено.
При h_nom система работает медленнее, но с нулевым перерегулированием σ = 0%,
что технологически допустимо: брака нет, просто регулирование чуть медленнее.

---

**Q: Что такое запас по фазе и почему он важен?**

A: Запас по фазе φ_зап = 180° + arg(W_ol(jωc)) — показывает, насколько далека
система от границы устойчивости. Здесь φ_зап = 71.7°. Практическое правило:
φ_зап ≥ 60° → монотонный переходный процесс, σ → 0%.
Если φ_зап < 30° → сильные колебания, система почти на грани.

---

**Q: Зачем датчик в обратной связи? Что это за звено?**

A: Датчик измеряет выходную мощность резания N2 и преобразует её в напряжение
для сравнения с заданием. Передаточная функция датчика W_датч(p) = Ku/(K3·p+1),
где Ku = 13.64·10⁻³ — коэффициент преобразования, K3 = 0.003 с — инерционность
самого датчика. K3 маленькое, поэтому датчик практически безынерционный.

---

**Q: Почему объект 3-го порядка, хотя объектов в схеме несколько?**

A: Порядок = количество полюсов передаточной функции = степень знаменателя.
Двигатель подачи имеет знаменатель 2-го порядка (T1·Tm·p² + Tm·p + 1).
Привод ШК — 1-го порядка (T2·p + 1). Итого 2+1 = 3.
Датчик тоже 1-го порядка, но он в обратной связи, не в прямой цепи.

---

**Q: В чём разница между дифференциальной эволюцией и частотным методом?**

A: Оба дают одинаковый результат (Ti = Tm, Td = T1), но:
- Диф. эволюция — численный перебор, объясняет «как нашли», но не «почему»
- Частотный метод — аналитика по ЛАЧХ, объясняет физический смысл параметров

В реальной практике сначала применяют аналитику (быстро, понятно), затем
численная оптимизация уточняет результат.

---

**Q: Что такое статическая точность δ = 0.6% и почему фактически δ = 0?**

A: δ — установившаяся ошибка при постоянном задании. Требование δ ≤ 0.6%
означает, что мощность резания должна поддерживаться с точностью 0.6%.
Фактически δ = 0, потому что в регуляторе есть I-канал (интегратор).
По теореме: система с интегратором в разомкнутом контуре имеет нулевую
статическую ошибку при постоянном воздействии — автоматически, без подбора Kp.
